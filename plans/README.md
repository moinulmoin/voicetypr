## Concurrency protocol (multiple agents/sessions work this repo)

1. **Before executing a plan**: set its row below to
   `IN PROGRESS — claimed <session/agent> <date>` and commit that one-line
   change first. A row already IN PROGRESS / NEEDS-SMOKE / DONE / REJECTED is
   CLAIMED or CLOSED — pick different work.
2. **NEEDS-SMOKE means code-frozen**: implementation is done, committed, and
   gate-green; only interactive desktop smoke remains (see `plans/SMOKE.md`).
   Never re-implement, "improve", or re-audit a NEEDS-SMOKE plan.
3. **Done/rejected plan files live in `plans/archive/`** — reference material
   only.
4. New plan files MUST take the next free number AND add a row here in the
   same commit, so concurrent sessions see the claim.

## Findings considered and rejected (do not re-audit)

- **Passwordless LAN sharing as a vulnerability** — by design: sharing is
  off by default, the UI labels the password "(Optional)"
  (`NetworkSharingCard.tsx:788`), and model control is hard-gated on a
  password. Optional hardening (default random token) available if the
  product stance changes.
- **`tauri-plugin-global-shortcut` (Rust) as dead dependency** — false:
  heavily used (`lib.rs`, `commands/settings.rs`, `commands/shortcuts.rs`,
  `recording/*`). The separate native-trigger rewrite lives on another
  branch. Only the *frontend* npm package is unused (handled in plan 009).
- **`sha1` crate as dead dependency** — false: active legacy checksum path
  (`whisper/manager.rs:425-432`). Real residue: the catalog field named
  `sha256` holds SHA1 values for built-in models (`manager.rs:96-99`).
  **Deferred**: rename to `checksum` + algorithm tag when the model catalog
  next changes; touches Rust + TS types + tests for cosmetic gain.
- **pnpm audit advisories (7 prod / 13 dev)** — all in dev-tooling chains
  (`shadcn` CLI, `conventional-changelog-cli`, `eslint-plugin-jsx-a11y`);
  nothing ships in the app bundle. Prod-profile noise eliminated by plan 009
  (shadcn → devDependencies).
- **Remote HTTP returns raw internal error strings** (SECURITY-04) — real,
  but the durable fix is the stable error taxonomy in plan 012's contract;
  patching strings piecemeal now would be churn.
- **`useRecording` async listener-cleanup race** — real but minor (the hook
  mounts once in a long-lived window). Standard disposed-flag fix; do it
  opportunistically next time the hook is touched. Not worth a plan.
- **Blocking recorder stop busy-polls 100 ms × 5 s on async threads** —
  real; deliberately NOT bundled into plan 008 (scope control). Candidate
  plan 013 if it shows up in practice (symptoms: UI freezes on stop with a
  wedged audio backend).
- **God-module splits** (`commands/audio.rs` 5.3k lines, `writing.rs` 3.4k,
  `commands/remote.rs`, `commands/settings.rs`) — deferred until plan 003's
  characterization net exists and plan 012 fixes the seams to split along.
  Splitting first would churn every other plan's line references.
- **Frontend state consolidation** (3 mechanisms: contexts, Zustand, raw
  Tauri events) — deferred; MED confidence on render cost, and the backend
  contract work dominates. Revisit after 012's implementation.
- **Mock-heavy frontend component tests** — acknowledged; add integration
  harnesses opportunistically per feature, not as a standalone refactor.
- **Stale deferred-items doc** (Voice Commands listed as future work but
  already shipped: `writing.rs:122-144`, `EnhancementSettings.tsx:411-424`,
  tests) — folded into plan 012's doc (its maintenance note flags the
